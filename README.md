# BSA8_UseCase
BSA08_UseCase

Проект по описанию вариантов использования (Use Cases) для системы онлайн-записи барбершопа (BRB).

## 🛠 Навыки
- Use Case Modeling
- Actor Analysis
- Flow of Events (Basic, Alternative)
- Include / Extend Relationships
- Preconditions & Postconditions

## 📋 Описание проекта
В ходе работы я разработала варианты использования для трёх ключевых ролей: менеджер, клиент и мастер. Описаны цели, предусловия, постусловия, потоки событий (основные и альтернативные), а также связи включения (include) и расширения (extend).

### Что именно было сделано:
1. **Use Case «Управление расписанием мастера» (менеджер)**: Описала UC для обеспечения актуального расписания и услуг мастера (BRB_UC_02). Разработала основной и альтернативные потоки (отмена, некорректные данные, конфликт записей).
2. **Use Case «Записаться на услугу» (клиент)**: Описала основной UC с целями, участниками, предусловиями, постусловиями (BRB_UC_101). Выделила включаемые UC «Выбрать услугу и мастера» (BRB_UC_102) и «Выбрать время записи» (BRB_UC_103), а также расширяющий UC «Зарегистрироваться в системе» (BRB_UC_104).
3. **Потоки для UC клиента (BRB_UC_101–104)**: Разработала детальные потоки событий (основной, альтернативные – нет слотов, выбор «любого мастера», некорректные данные, отмена, ошибка уведомления).
4. **Use Case «Просмотреть расписание и записи» (мастер)**: Описала UC для просмотра мастером своего расписания и записей (BRB_UC_201). Выделила включаемый UC «Получить детали записи» (BRB_UC_203) и расширяющий UC «Просмотреть отзывы» (BRB_UC_202).
5. **Потоки для UC мастера (BRB_UC_201–203)**: Разработала основной и альтернативные потоки (просмотр деталей записи, переключение периода, фильтрация отзывов, возврат к расписанию).
